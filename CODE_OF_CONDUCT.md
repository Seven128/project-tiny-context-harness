# Code of Conduct

Project Tiny Context Harness is a small open source developer-tool project. The project works best when issues, pull requests, launch feedback and adoption reports stay specific, respectful and useful to maintainers and contributors.

## Expected Behavior

- Be respectful of different experience levels, tools, languages and workflows.
- Keep criticism focused on the project, code, documentation, evidence or tradeoffs.
- Assume public discussions may be read later by new contributors trying to understand the project.
- Be clear when a comment is personal preference, product feedback, benchmark evidence or a security concern.
- Respect privacy. Do not post private repository details, customer data, private chat logs, secrets, tokens or personal contact details.

## Unacceptable Behavior

- Harassment, threats, personal attacks, slurs or sustained hostile comments.
- Publishing private information without explicit permission.
- Repeated off-topic promotion, spam or demands for support outside the project scope.
- Misrepresenting benchmark, adoption, security, award or maintainer claims.
- Using issues, pull requests or discussions to pressure contributors into public conflict.

## Scope

This Code of Conduct applies in project spaces such as GitHub issues, pull requests, discussions, release comments and official project launch threads. It also applies when someone is clearly representing Project Tiny Context Harness in public.

## Reporting

If the issue is public and low-risk, comment in the thread and ask for the behavior to stop or ask a maintainer to moderate.

If the report is sensitive, do not include private details in a public issue. Open a GitHub issue titled `Conduct contact needed` with no incident details, or use GitHub's platform-level abuse/reporting tools when appropriate. The maintainer will arrange a private contact path when one is needed.

Security vulnerabilities should follow [SECURITY.md](SECURITY.md), not this conduct process.

## Enforcement

Maintainers may:

- ask for edits or clarification,
- hide, edit or delete comments when platform rules allow it,
- lock heated or off-topic conversations,
- close issues or pull requests that stop being actionable,
- block or report accounts for serious or repeated violations.

The project is currently single-maintainer, so enforcement is intentionally small and practical. Decisions should protect contributors, preserve useful technical discussion and keep project claims honest.

## Attribution

This is a short project-specific Code of Conduct. It is informed by common open source community practices and GitHub community health guidance, but it is not a verbatim copy of Contributor Covenant.
