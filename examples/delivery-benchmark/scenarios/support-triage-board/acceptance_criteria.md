# Acceptance Criteria

- AC-SUP-001: API can create and list tickets.
- AC-SUP-002: Board groups tickets by `new`, `triaged`, `waiting` and `resolved`.
- AC-SUP-003: Priority score uses customer tier, age and channel.
- AC-SUP-004: Enterprise tickets older than 2 hours show SLA risk.
- AC-SUP-005: Assignment and status movement are persisted.
- AC-SUP-006: UI has loading, empty and error states.
- AC-SUP-007: API errors are structured.
- AC-SUP-008: Tests include API regression and UI/browser smoke.
- AC-SUP-009: UI/UX handoff describes screen states and non-visual accessibility expectations.
- AC-SUP-010: TESTING bugfix route is recorded if final testing finds a behavior mismatch.
