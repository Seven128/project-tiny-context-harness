## 4. Defect Candidates

| ID | Candidate | Evidence | Impact |
|---|---|---|---|
| `LAB-001` | Package materializes a Makefile that calls `tools/*.py`, but `tools/**` is not distributed into consumer repositories. | `make validate-harness` failed with `can't open file .../tools/validate_harness.py`. | Documented `make validate-*`, `make status`, and `make docs-overview` commands do not work for installed consumers. |
| `LAB-002` | Documented lifecycle transition command depends on `tools/transition.py`, which is not distributed. | `python3 tools/transition.py --to ARCHITECTING` failed with missing file. | Consumers cannot follow documented phase transition workflow from package assets alone. |
| `LAB-003` | Package CLI does not expose validators for `validate-review`, `validate-test`, `validate-release`, or `validate-rfc`. | `npx sdlc-harness validate validate-review` and siblings returned `unknown validator`. | Consumers cannot use CLI fallback for later-stage gates when Makefile tools are unavailable. |
| `LAB-004` | Package has no packaged docs-overview command. | `make docs-overview` failed with missing `tools/build_doc_overviews.py`; no CLI equivalent exists. | Consumers cannot regenerate generated `overview.md` using package-only assets. |

## 5. Gate Result

- Decision: BLOCKED for full documented consumer workflow.
- Passing subset: package install, init/sync/upgrade/doctor, managed assets, local override handling, PM/design/dev CLI validators, task protocol validators, and parallel contract validators.
- No workaround copied source-repo `tools/**` into the lab.
- Recommended next step: create a dedicated DEV task to either distribute the needed tools, replace generated Make targets with `npx sdlc-harness` commands, or expand CLI coverage so installed consumers can run the documented workflow end to end.
